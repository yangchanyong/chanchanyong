1. 대규모 서비스 리빌딩 및 클라우드 마이그레이션

뷰티 아카데미 / IT 아카데미

기존 온프레미스(IDC) 환경에서 운영되던 레거시 웹사이트를 클라우드 환경으로 이관하고, 사용 기술을 현대화하여 서비스 안정성을 확보했습니다.

  • 언어 전환: Classic ASP로 작성된 기존 코드를 PHP 기반으로 전면 재작성하여 유지보수성 및 시스템 성능 향상
  • 인프라 이관: IDC 서버 환경을 AWS 인프라로 성공적으로 마이그레이션하여 트래픽 대응력 강화
  • 운영 개선 성과: 유연한 클라우드 환경(Scale-out) 구축을 통해 트래픽 피크 타임의 서비스 지연 문제를 해소하고, 레거시 코드 제거로 신규 기능 배포에 드는 공수를 대폭 단축함

도입 배경: 기존 IDC 환경의 노후화된 서버와 스케일아웃 불가능 문제 해결을 위해 AWS 및 최신 언어(PHP) 전환 결정.

트러블슈팅: 대용량 데이터 이관 중 타임아웃 발생 -> Chunk 단위 분할 비동기 처리 스크립트 작성 및 트랜잭션 로그 후속 동기화로 무중단 마이그레이션 완수.

회고: 데이터 이관 시 발생하는 엣지 케이스를 경험하며 데이터 정합성과 무중단 배포의 중요성을 깨달음.

2. CI/CD 배포 파이프라인 고도화 및 이원화 운영

기존 Jenkins 기반의 배포 파이프라인 성능을 개선하기 위해, 개발 서버 환경에 GitLab CI/CD를 선제적으로 도입하여 배포 속도를 최적화했습니다.

  • 파이프라인 최적화: 기존 Jenkins 빌드 병목을 해소하고자 개발 서버용 GitLab Runner 구축 및 파이프라인(.gitlab-ci.yml) 마이그레이션
  • 하이브리드 운영: 안정성을 위해 실서버(Jenkins)와 개발서버(GitLab CI/CD)의 이원화된 파이프라인 환경 구축 및 운영
  • 배포 소요 시간 단축: 캐싱 최적화를 통해 개발 서버 배포 시간 3분 -> 10초로 단축

도입 배경: 개발/실서버 모두 Jenkins를 사용 중이었으나, 개발 서버의 긴 배포 시간으로 인한 피드백 지연을 해결하기 위해 GitLab CI/CD 선제 도입.

트러블슈팅: 이관 초기 매 빌드마다 의존성을 재다운로드하여 속도 저하 -> GitLab CI의 cache/artifacts 기능을 활용한 파이프라인 스크립트 최적화로 배포 시간 획기적 단축(3분->10초).

회고: 시스템 안정성을 위해 개발 서버부터 점진적으로 도입하는 투트랙 전략의 유효성과 파이프라인 최적화가 팀 생산성에 미치는 영향을 체감함.

3. 레거시 시스템 모던화 (Modernization)

사내 및 서비스 중인 레거시 시스템의 프레임워크 전환 및 아키텍처 개선을 수행했습니다.

  • 프레임워크 도입: 유지보수가 까다로운 Classic PHP 시스템을 CodeIgniter 프레임워크로 성공적으로 마이그레이션
  • 구조 개선: 코드 구조 개선 및 웹사이트 페이지 리뉴얼 작업을 주도하여 사용자 경험(UX) 개선

도입 배경: 절차지향적 코드로 인한 사이드 이펙트 방지를 위해 MVC 패턴을 강제하는 CodeIgniter 도입.

트러블슈팅: 무분별한 전역 변수 충돌 -> 의존성 주입(DI) 활용 및 싱글톤 패턴 클래스로 묶어 중앙 통제하도록 리팩토링.

회고: 기존 기능을 100% 보장하며 내부 구조를 개선하는 과정에서 테스트 코드의 중요성을 절감함.

4. 공통 기능 모듈화 및 시스템 아키텍처 설계

파편화된 비즈니스 로직을 통합하고, 향후 확장성을 고려한 백엔드 구조를 설계했습니다.

  • 모듈화: 소셜 로그인, 고객 상담, 결제 연동 기능을 독립적인 모듈로 개발하여 타 프로젝트에서의 재사용성 100% 확보
  • 데이터 파이프라인: 마케팅 분석 시스템의 기반 데이터를 안정적으로 적재하고 가공하는 데이터 파이프라인 기초 구축

도입 배경: 신규 프로젝트마다 반복되는 개발 비효율 개선 및 유지보수 포인트 통합을 위해 핵심 로직 캡슐화.

트러블슈팅: 각 서비스별 결제 후처리 로직 상이 문제 -> 이벤트 기반(Event-Driven) 웹훅 구조로 설계하여 결제 모듈의 단일 책임을 유지하고 서비스별 구독 처리하도록 개선.

회고: 객체지향의 SRP(단일 책임 원칙)와 OCP(개방-폐쇄 원칙)의 중요성을 실무 레벨에서 깊이 이해함.
